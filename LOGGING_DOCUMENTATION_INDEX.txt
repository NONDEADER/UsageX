## Critical Information at a Glance

### Storage Structure
`
browser.storage.local:
  debug_logs: [
    { timestamp: "14:35:42.123", sender: "background", 
      level: "debug", message: "..." },
    ...
  ]
  debug_mode_until: <unix_timestamp_ms>
`

### Log Lifecycle
1. RawLog() called from code
2. Check debug mode and log level
3. Create log entry with timestamp
4. Retrieve existing logs array
5. Append new entry
6. Enforce 1000-entry limit (FIFO)
7. Save back to storage

### UI Update Cycle
1. showLogs() fetches all logs
2. Clear container, rebuild DOM
3. Color code by sender and level
4. Auto-scroll to bottom
5. Repeat every 5 seconds (if enabled)

### Debug Mode States
- Disabled: No logging unless FORCE_DEBUG
- Temporary: Enabled for 1 hour
- Permanent: No expiration until manually disabled
