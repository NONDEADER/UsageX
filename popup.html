UsageX v2
Real-time Claude.ai token tracker · Local only · No network calls
Dashboard
Session bar 5h window

Tracks how much of your rolling 5-hour token window you've used. Hover the bar to see raw token counts.

Weekly bar 7-day window

Tracks usage across the 7-day token window. Resets on Claude's own rolling schedule.

Rate indicator dot

The pulsing dot next to the session % shows your token burn rate per hour.

Teal Below normal · <19%/h · great pace
Gray On track · ~20%/h
Amber Above normal · >21%/h
Orange Burning fast · ≥30%/h · slow down!
Live dot (header)

Shows server load in real time.

Teal Off-peak · limits deplete normally
Red Peak hours · 6:30 PM – 12:30 AM IST · limits deplete faster
Sidebar Panel
Peak-hours clock

24-hour timeline strip highlighting 6:30 PM – 12:30 AM IST — when Claude's shared infrastructure is busiest.

Messages remaining

Estimated messages left in the current 5-hour window, based on your average token cost per message.

Settings (⚙ in sidebar)
Sidebar side
Dock the panel left or right of Claude's nav sidebar.
Floating mode
Detaches the panel — drag the header to reposition freely.
Opacity
Adjust floating-panel transparency from 10% to 100%.
Resizable
Enables drag handles to resize the floating panel freely.
Timezone
Override timezone for reset-time display (defaults to browser timezone).
Debug logging
Toggle verbose logging. View in Tools → Debug Logs or export as JSON.
Keyboard Shortcuts
Toggle sidebar panel Alt+U
Token Estimates
Counts are approximate and labelled ~est. Claude doesn't expose exact counts in the browser — input length is estimated from char count (÷ 4) and thinking tokens per effort level.
Privacy & Data Storage
Stored in a local, private browser database (IndexedDB) on your computer
Your data is never uploaded, shared, or sent to any external server
Offline database is secure and sandboxed — websites cannot access it
Zero analytics, trackers, cookies, or telemetry are used in this extension
