Privacy Policy

Last updated: July 2026  ·  Applies to UsageX v2 (Firefox & Chrome extension)

Short version: UsageX stores all your usage data locally on your device. The only data that ever leaves your device is what you voluntarily submit via the feedback form.

1. What data is collected

UsageX tracks your Claude.ai usage statistics and stores them entirely within your browser's local storage (browser.storage.local and IndexedDB). No account is created, no server is involved, and no data is synced to any cloud service.

Data stored locally includes:

  • Session and weekly token usage percentages (read from Claude's own API responses)
  • Daily message count, active time, estimated token totals
  • Effort breakdown (Low / Medium / High / Max thinking levels)
  • Models used per session
  • Conversation-level message and token estimates
  • Reset timestamps for session and weekly limits
  • Your Claude plan (Free / Pro / Max), name, and email — scraped from the Claude UI or API response and used only to label the dashboard
  • A rolling window of one-way djb2 hashes of outgoing messages, used exclusively to prevent double-counting (the original message text is never stored)
  • Extension settings (sidebar position, notification preferences, timezone, etc.)

2. Data that leaves your device

Scenario Data sent Destination Trigger
Feedback form Feedback type, message text, optional email address, optional diagnostics snapshot Google Sheets via Google Apps Script (developer-controlled) Manual Only when you click "Send Feedback"
Everything else Nothing — Never

The feedback form is entirely optional. Submitting it is always a deliberate, user-initiated action. The form is never triggered automatically.

3. Feedback form — detailed breakdown

When you submit the feedback form, the following information is sent in a single HTTPS POST request to a Google Apps Script endpoint operated by the developer:

  • Feedback type — Bug, Feature, or Question (you select this)
  • Message — the text you typed
  • Email — only if you choose to enter one (the field is optional; it defaults to "Anonymous")
  • Diagnostics snapshot — only if the "Diagnostics info" toggle is checked (on by default). This includes: extension version, browser user agent string, your Claude plan, timezone setting, today's aggregate message/token counts (numbers only — no message content), and a count of stored debug log entries.

This data is stored in a private Google Spreadsheet visible only to the developer and is used solely to improve UsageX. It is not sold, shared, or used for advertising.

5. Data retention and deletion

All locally stored data can be deleted at any time from within the extension:

  • Reset Stats — clears today's counters and the 30-day history
  • Clear Logs — deletes all debug log entries
  • Uninstalling the extension removes all local storage automatically

Data you submitted via the feedback form cannot be retrieved or deleted after submission, as it is stored in the developer's private spreadsheet with no user identifiers (unless you provided an email address, in which case you can request deletion by emailing nondeader.dev@gmail.com).

6. Third-party services

UsageX does not use any analytics services, crash reporting tools, or advertising networks. The only third-party service the extension communicates with is Google Apps Script, and only when you explicitly submit feedback.

7. Children

UsageX is not directed at children under 13 and does not knowingly collect information from children.

8. Changes to this policy

If the privacy practices of UsageX change significantly (e.g. new data collection is introduced), this page will be updated and the "Last updated" date will change. Continued use of the extension constitutes acceptance of the updated policy.

9. Contact

Questions or requests regarding this privacy policy can be sent to nondeader.dev@gmail.com.
